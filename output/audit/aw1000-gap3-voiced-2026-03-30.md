## 1. State Machine

```
                    +--------------------------------------------------+
                    |           voiced Call State Machine               |
                    +--------------------------------------------------+

    +-------+
    | IDLE  |<---------------------------------------------------------+
    +---+---+                                                          |
        |                                                              |
        +--- RING URC -----> +---------+                               |
        |                    | RINGING |--- NO CARRIER --> [IDLE]      |
        |                    +----+----+                               |
        |                         |                                    |
        |                   offhook (ATA)                              |
        |                         |                                    |
        |                    +----v-------+                            |
        |                    | OFFHOOK_IN |--- NO CARRIER/onhook --+  |
        |                    +------------+                         |  |
        |                                                           |  |
        +--- offhook ------> +----------+                           |  |
                              | DIALTONE |                          |  |
                              +----+-----+                          |  |
                                   |                                |  |
                              DTMF digit                            |  |
                                   |                                |  |
                              +----v----+                           |  |
                              | DIALING |--- onhook --> [IDLE]      |  |
                              +----+----+                           |  |
                                   |                                |  |
                          timeout or '#'                             |  |
                                   |                                |  |
                          +--------v------+                         |  |
                          | OFFHOOK_OUT   |--- NO CARRIER/onhook --+  |
                          +---------------+                         |  |
                                                                    |  |
                          +--------+                                |  |
                          | HANGUP |<-------------------------------+  |
                          +---+----+                                   |
                              | 500ms                                  |
                              +----------------------------------------+
```

**States:**
- **IDLE** — Phone on-hook, no call active. Monitoring for RING URCs and hook events.
- **RINGING** — Incoming call detected (RING/+CRING URC). SLIC ring generator active.
- **OFFHOOK_IN** — Incoming call answered (ATA sent). Voice path active.
- **DIALTONE** — User went off-hook from idle. Dial tone playing. Waiting for DTMF.
- **DIALING** — Collecting DTMF digits. 4s inter-digit timeout, min 3 digits.
- **OFFHOOK_OUT** — Outgoing call placed (ATD sent). Ringback tone until connect.
- **HANGUP** — Transitional state. Plays busy tone briefly, then returns to IDLE.

**Transitions:**
| From | Event | To | Action |
|------|-------|----|--------|
| IDLE | RING URC | RINGING | SLIC ring on |
| IDLE | offhook | DIALTONE | Play dial tone |
| RINGING | offhook | OFFHOOK_IN | Ring off, send ATA |
| RINGING | NO CARRIER | IDLE | Ring off |
| DIALTONE | DTMF digit | DIALING | Stop dial tone, start collecting |
| DIALING | timeout/# | OFFHOOK_OUT | Send ATD, play ringback |
| DIALING | onhook | IDLE | Send ATH |
| OFFHOOK_OUT | +CIEV:call,1 | OFFHOOK_OUT | Stop ringback (connected) |
| OFFHOOK_OUT | NO CARRIER/onhook | HANGUP | Send ATH, play busy |
| OFFHOOK_IN | NO CARRIER/onhook | HANGUP | Send ATH, play busy |
| HANGUP | 500ms timer | IDLE | Stop all tones |

## 2. ioctl Interface

The SLIC driver (`slic-si3226x.c`) was extended with a misc char device interface
exposing `/dev/proslic0` and `/dev/proslic1` (one per SLIC channel).

| ioctl | Direction | Description |
|-------|-----------|-------------|
| `PROSLIC_RING_ON` | none | Activate ring generator (LF_RINGING) |
| `PROSLIC_RING_OFF` | none | Deactivate ring, return to FWD_ACTIVE |
| `PROSLIC_GET_HOOK_STATUS` | read | Returns 1=offhook, 0=onhook |
| `PROSLIC_GET_DTMF` | read | Returns decoded DTMF digit (0-15) or -1 |
| `PROSLIC_PLAY_TONE` | write | Play tone: 0=none, 1=dial, 2=ringback, 3=busy, 4=congestion |
| `PROSLIC_SET_LINEFEED` | write | Set arbitrary linefeed mode (LF_OPEN..LF_RING_OPEN) |

Magic: `'P'`, ioctl numbers 1-6 using `_IO`/`_IOR`/`_IOW` macros.

**Implementation notes:**
- Hook status read from Si3226x register 4 (LOOP_STAT), bit 0
- DTMF decoded from register 24 (TONDTMF), valid bit 4, digit bits 0-3
- Tone generation via oscillator control registers 48-49 (OCON/OMODE)
- IRQ handler wakes poll() waiters on hook state change events

## 3. AT Commands

### Initialization sequence (sent at daemon start):
1. `ATE0` — Disable echo
2. `AT+CMEE=2` — Verbose error reporting
3. `AT+CRC=1` — Extended format ring (+CRING: VOICE)
4. `AT+CLIP=1` — Caller ID presentation
5. `AT+QCFG="ims",1` — Enable IMS/VoLTE
6. `AT+QCFG="ims/callwaiting",1` — Enable call waiting over IMS
7. `AT+CVHU=0` — ATH disconnects voice calls

### URCs parsed:
- `RING` / `+CRING:` — Incoming call indication
- `NO CARRIER` — Call disconnected
- `+CIEV: call,0` / `+CIEV: "call",0` — Call ended
- `+CIEV: call,1` / `+CIEV: "call",1` — Call connected
- `+CLIP:` — Caller ID (logged)

### Commands sent during call:
- `ATA` — Answer incoming call
- `ATH` — Hang up
- `ATD<digits>;` — Dial (voice call, semicolon = voice)

## 6. Known Limitations

1. **DTMF via SLIC hardware** — Si3226x DTMF detection register addresses (reg 24)
   are based on datasheet; actual register map may differ per silicon revision.
   Must verify on real hardware.

2. **Tone frequencies** — Oscillator registers 48/49 are used with preset mode
   values. Actual tone frequencies depend on ProSLIC API preset configuration
   in `si3226x_userdef.c`. The register writes enable/disable oscillators but
   frequency parameters must be loaded during init.

3. **Single channel** — voiced only manages `/dev/proslic0` (channel 0).
   Two-line operation would require a second daemon instance or multi-channel support.

4. **No call waiting** — While AT+QCFG enables IMS call waiting, the state machine
   does not handle a second incoming call during an active call.

5. **No emergency call** — No special handling for emergency numbers (112/911).

6. **No codec negotiation** — Voice codec is fixed by PCM hardware configuration
   (8-bit linear PCM). AMR-WB VoLTE codec transcoding happens in the modem.

7. **SIGHUP reopen** — AT port reopen resets modem init but does not reset
   in-progress call state. A call in progress during SIGHUP may become orphaned.

## 8. Remaining Risks

1. **ProSLIC API availability** — The driver depends on proprietary Si3226x API source
   files that must be obtained separately. Without these, the kernel module will not
   compile. The DTMF detection and tone generation register addresses used in the ioctl
   layer bypass the API and access registers directly, which may not be sufficient.

2. **PCM clock source** — The Si3226x on the AW1000 may provide external PCLK
   (as suggested by GPIO36 pcm_pclk being an input). If the IPQ8074 PCM controller
   expects to be the clock master, there may be a clock conflict.

3. **IRQ handling** — The current IRQ handler simply wakes poll waiters. A production
   driver should read the SLIC interrupt status register to distinguish between hook
   change, DTMF ready, ring trip, and fault interrupts.

4. **SPI concurrency** — The ioctl path reads SLIC registers outside the ProSLIC API
   context. While Linux SPI bus locking prevents bus conflicts, the ProSLIC API
   internal state is not protected against concurrent access.

5. **Modem AT port contention** — If other processes (e.g., ModemManager, qmodem)
   also open `/dev/ttyUSB2`, AT command responses may be interleaved. The voiced
   daemon should be the exclusive user of the AT port, or a multiplexing layer
   (e.g., `cmux`) should be used.
